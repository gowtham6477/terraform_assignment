📖 What Terraform Created

A breakdown of the core infrastructure components instantiated dynamically on AWS using declarative HCL files:

VPC (Virtual Private Cloud) ▶
A logically isolated virtual network inside AWS. It defines your own IP address range (10.0.0.0/16), subnets, route tables, and network gateways. It serves as the foundational boundary for security and network topology.
Public Subnet ▶
A segmented range of IP addresses within the VPC (10.0.1.0/24). By associating this subnet with a route table that directs external traffic to the Internet Gateway, resources in this subnet can acquire public IP addresses and communicate with the outside world.
Security Group (Firewall) ▶
Acts as a virtual firewall for the EC2 server, controlling inbound and outbound traffic. It is configured to allow inbound HTTP (port 80) from anywhere, restricted SSH (port 22) for administrator access, and all outgoing traffic to download packages.
EC2 (Elastic Compute Cloud) Instance ▶
A virtual machine in the cloud. It runs a t3.micro instance type in eu-north-1 (Stockholm), backed by Amazon Linux 2023. Upon creation, a bootstrap script (user_data) automatically downloads, configures, and launches Apache to serve this showcase application.
S3 (Simple Storage Service) Bucket ▶
An industry-standard object storage resource. Highly scalable, durable, and secure, it is created with a globally unique name: __BUCKET_NAME__. It is commonly used for file uploads, backups, or hosting frontend client builds.
